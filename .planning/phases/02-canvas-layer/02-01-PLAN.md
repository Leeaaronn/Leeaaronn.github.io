  <action>
Replace the stub in src/stars.js with a full Three.js star field implementation:

1. Import from 'three': Scene, PerspectiveCamera, WebGLRenderer, BufferGeometry, Float32BufferAttribute, PointsMaterial, Points

2. Export function `initStars()` that:
   a. Creates a WebGLRenderer with `{ alpha: true, antialias: false }` — antialias false because points don't benefit from it
   b. Sets renderer size to `window.innerWidth` x `window.innerHeight`
   c. Sets `renderer.domElement.style.position = 'fixed'`
   d. Sets `renderer.domElement.style.top = '0'`
   e. Sets `renderer.domElement.style.left = '0'`
   f. Sets `renderer.domElement.style.zIndex = '0'` — behind everything
   g. Sets `renderer.domElement.style.pointerEvents = 'none'` — non-interactive
   h. Sets `renderer.setPixelRatio(Math.min(window.devicePixelRatio, 2))` — RESP-02 cap
   i. Appends `renderer.domElement` to `document.body`

3. Creates a PerspectiveCamera with FOV 60, aspect ratio window.innerWidth/window.innerHeight, near 0.1, far 1000. Position camera at z=1.

4. Creates a BufferGeometry with 3000 random star positions:
   - Each star x, y, z is `(Math.random() - 0.5) * 2` — distributed in a cube around origin
   - Set as a Float32BufferAttribute with item size 3

5. Creates a PointsMaterial with `{ color: 0xffffff, size: 0.003, sizeAttenuation: true }`

6. Creates a Points mesh from the geometry and material, adds to scene

7. Creates animation loop using `renderer.setAnimationLoop(animate)`:
   - In `animate()`: rotate the Points mesh slowly — `points.rotation.y += 0.0001` and `points.rotation.x += 0.00005` for subtle drift (STAR-02)
   - Call `renderer.render(scene, camera)`

8. Adds a window resize handler:
   - Updates camera aspect ratio
   - Updates camera projection matrix
   - Resizes renderer

The canvas should have NO background color (alpha: true) so the black body background shows through.
